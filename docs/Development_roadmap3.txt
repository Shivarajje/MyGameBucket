# Build Order

The implementation should follow a strict order.

Building features out of order increases complexity.

Recommended order:

```text id="roadmap03"
Project Setup

↓

Authentication

↓

Database

↓

Game Service

↓

Game Search

↓

Game Pages

↓

Library

↓

Profile

↓

Journal

↓

Collections

↓

User Search

↓

Admin

↓

Polish

↓

Deployment
```

This minimizes blockers and dependency issues.

# Development Priorities

Highest Priority

* Stable Architecture
* Correct Database Design
* Clean APIs
* Modular Services

Medium Priority

* Visual Polish
* Microinteractions
* Animations

Lower Priority

* Cosmetic improvements
* Additional themes
* Minor UI refinements

Correctness always takes priority over appearance.

---

# Definition of a Finished Milestone

A milestone is complete only when:

✓ Requirements implemented

✓ Documentation updated

✓ Responsive

✓ Accessible

✓ Error handling complete

✓ Tested

✓ Reviewed

✓ Ready for production

A partially implemented milestone is not considered complete.

---

# Parallel Development

Once the foundation is complete, several areas can be developed simultaneously.

Example:

```text id="roadmap04"
Library

Profile

Journal

Collections
```

Each feature is isolated through the Service Layer, making parallel development safe.
