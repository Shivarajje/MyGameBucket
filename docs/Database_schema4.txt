# Table 4 — journal_entries

## Purpose

The `journal_entries` table stores a player's personal memories for each game.

Unlike reviews, these entries are intended to capture personal experiences, emotions, memorable moments, or milestones during a player's journey.

Each user may create **one Journal Entry per game**.

---

## Philosophy

Journal Entries are personal.

They are **not** reviews.

They are **not** comments.

They are **not** public discussions.

They exist to preserve a player's memories associated with a specific game.

# Performance Notes

Expected workload:

* Moderate reads
* Moderate writes

* User Profiles
* Game Pages
* Personal Library

Simple indexing is sufficient.

---

# Design Notes

Journal Entries intentionally avoid becoming a review system.

The focus is:

"My experience."

rather than

"My score."

This distinction supports the core vision of MyGameBucket as a gaming journal rather than another review platform.

# Table 5 — collections

## Purpose

Collections allow users to organize games into custom groups.

Examples:

* Masterpieces
* Horror Games
* Childhood Favorites
* Games to Replay
* Best Co-op Games

Collections contain metadata only.

Games are linked through the `collection_games` table.
