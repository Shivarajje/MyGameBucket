# Table 6 — collection_games

## Purpose

The `collection_games` table creates the many-to-many relationship between collections and games.

A collection may contain many games.

A game may belong to many collections.

This table stores only the relationship.

No duplicate game information is stored.

# Unique Constraints

Prevent duplicate games.

Constraint

```text id="0lwzk0"
(collection_id, game_id)

UNIQUE
```

A collection cannot contain the same game twice.


# Design Notes

This table intentionally stores only relationships.

No game metadata belongs here.
