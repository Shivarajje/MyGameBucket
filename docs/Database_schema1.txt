# Purpose

This document defines the complete production database schema for MyGameBucket.

Unlike the Database Architecture document, this document specifies the actual database structure that should be implemented inside Supabase.

Every table includes:

* Purpose
* Columns
* Constraints
* Relationships
* Indexes
* RLS Policies
* Validation Rules
* Future Expansion
* Performance Notes
* Example Record

This document acts as the **single source of truth** for the database schema.

Whenever the schema changes:

1. Update this document.
2. Generate SQL migration.
3. Apply migration.

Never modify the database without first updating this document.

# Table 1 — profiles

## Purpose

Stores public profile information for every authenticated user.

Each authenticated user owns exactly one profile.

Authentication information remains inside Supabase Auth.

The profile contains only application-specific information.

## Performance Notes

The profiles table is expected to be one of the most frequently queried tables.

Queries include:

* User Search
* Profile Pages
* Public Profiles
* Favorites

Indexes on username and visibility are essential.
