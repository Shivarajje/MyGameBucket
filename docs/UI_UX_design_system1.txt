# Purpose

This document defines the complete visual language of MyGameBucket.

Its purpose is to ensure every page, component and interaction feels like part of one cohesive application.

The design system should make the website feel:

* Premium
* Cinematic
* Modern
* Elegant
* Fast
* Consistent

Every visual decision should follow this document.

---

# Design Philosophy

MyGameBucket is **not** a gaming news website.

It is **not** a game store.

It is **not** a review website.

It is a personal gaming journal.

Everything should reinforce that feeling.

The interface should make players feel like they are browsing their own gaming history.

---

# Design Principles

Every UI decision should follow these principles.

---

## Principle 1 — Content First

Games are the focus.

The interface should support the content,

never compete with it.

---

## Principle 2 — Consistency

Buttons,

Cards,

Spacing,

Typography,

Animations,

Colors,

Interactions

should behave consistently everywhere.

Users should learn the interface once.

---

## Principle 3 — Cinematic

The application should feel premium.

Not flashy.

Not overwhelming.

Subtle lighting,

depth,

motion,

and typography create immersion.

---

## Principle 4 — Fast

Beautiful interfaces should never feel slow.

Animations must support speed,

not reduce it.

---

## Principle 5 — Accessibility

Beauty should never reduce usability.

Everything should remain readable,

navigable,

and responsive.

---

# Visual Identity

Keywords

* Modern
* Elegant

* Excessive gradients
* Visual clutter
* Cartoon aesthetics

# Color Philosophy

Colors are used to create atmosphere.

Not decoration.

Most pages remain neutral.

Accent colors are introduced through:

* Genre
* Game
* Profile Theme
